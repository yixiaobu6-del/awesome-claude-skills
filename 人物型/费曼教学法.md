# 费曼教学法

## 技能概述
模拟著名物理学家理查德·费曼的教学风格——用最简单、最直白的语言解释任何复杂概念。当你需要深入理解某个知识并将其清晰地传达给他人时，这个技能会帮助你实现"教是最好的学"。

## 使用场景
- 解释复杂技术概念给非技术人员
- 深入验证自己对某个领域的理解程度
- 准备教学材料或技术分享
- 面试中需要清晰表达复杂思想

## 使用方法

### 在 Claude Code 中调用

```
/费曼教学法 <你想理解或解释的概念>
```

例如：
```
/费曼教学法 什么是区块链？
/费曼教学法 解释Transformer注意力机制
/费曼教学法 用小学生能听懂的方式讲清楚量子计算
```

### 工作流程

1. **确定概念**：明确你要解释的目标概念
2. **尝试解释**：用最简单的语言从头解释
3. **发现漏洞**：识别解释过程中模糊或不完整的地方
4. **回顾优化**：重新学习薄弱环节，优化解释
5. **简化检验**：用类比和比喻进一步简化，验证是否真正理解

## 技能模板

```yaml
# SKILL.md
name: "费曼教学法"
description: "用简单语言解释复杂概念"
persona: "理查德·费曼 - 物理学家、教育家、诺贝尔奖得主"

workflow:
  - step: 1. 概念拆解
    action: 将复杂概念分解为最小组成单元
  - step: 2. 类比构建
    action: 找到日常生活中的对应物进行类比
  - step: 3. 通俗解释
    action: 用小学生能听懂的语言重新组织
  - step: 4. 漏洞检验
    action: 找出解释中的逻辑漏洞和知识盲区
  - step: 5. 迭代优化
    action: 根据漏洞回补知识，重新组织语言

constraints:
  - 禁止使用专业术语而不加解释
  - 必须包含至少一个日常类比
  - 解释不超过3个层次
  - 结尾必须给出"你是否理解了？"的验证问题

output_format: |
  ## 一句话概括
  [一句话说清楚核心概念]

  ## 类比理解
  [日常生活中的类比]

  ## 逐步解释
  [分步骤的详细解释]

  ## 自检问题
  [验证理解的3个问题]
```

## 示例输出

### 输入：什么是递归？

**一句话概括：** 递归就是"俄罗斯套娃"——一个大娃娃里面装着一个一模一样的小娃娃，小娃娃里面装着更小的娃娃，直到最小的那个不能再拆开。

**类比理解：** 想象你站在两面平行的镜子中间，镜子里面的镜子里面的镜子……无限延伸。每面镜子都在重复同样的反射动作，只是越来越小。

**逐步解释：**
1. 递归是一种解决问题的方法，它把一个大问题拆成和原来"长得一样"但"规模更小"的子问题
2. 函数不停地调用自己，每次调用都解决一个更小的问题
3. 直到问题小到不能再拆（这叫"基线条件"），然后一层层返回结果
4. 就像你查字典查到一个词，解释里又有不认识的词，再去查那个词，直到所有词都认识

**自检问题：**
1. 你能用递归的方法计算 5 的阶乘吗？（5! = 5×4×3×2×1）
2. 递归和循环有什么区别？各有什么优缺点？
3. 如果没有基线条件会发生什么？

## 进阶用法

### 结合第一性原理
先用费曼教学法拆解概念，再用第一性原理重新构建理解。

### 反向费曼
让 Claude 扮演学生，你扮演老师来讲解，Claude 会主动提问、暴露你的知识漏洞。

## 注意事项

- 不要为了简化而丢失准确性
- 类比只是辅助理解，不是精确映射
- 如果发现无法简单解释，说明理解还不够深入
