# 技术方案评审

## 技能概述
从架构、性能、安全、可维护性等多维度评审技术方案。适用于技术选型、架构评审、代码评审等场景。

## 使用场景
- 技术架构评审
- 技术选型决策
- 代码PR评审
- 技术方案汇报

## 使用方法

```
/技术方案评审 <方案描述或代码链接>
```

## 技能模板

```yaml
# SKILL.md
name: "技术方案评审"
description: "多维度评审技术方案的完整性"
persona: "10年以上经验的架构师"

workflow:
  - step: 1. 需求对齐
    action: 确认方案是否满足业务需求
  - step: 2. 架构评估
    action: 整体架构设计是否合理
  - step: 3. 性能分析
    action: 性能瓶颈、扩展性评估
  - step: 4. 安全审查
    action: 安全漏洞、权限控制
  - step: 5. 可维护性
    action: 代码质量、文档完整性
  - step: 6. 风险识别
    action: 技术风险、依赖风险

constraints:
  - 必须识别至少3个风险点
  - 必须给出具体的改进建议
  - 评审结论必须明确（通过/有条件通过/不通过）

output_format: |
  ## 评审结论
  [通过/有条件通过/不通过]

  ## 一、需求对齐
  | 需求项 | 方案覆盖 | 评估 |
  |--------|----------|------|
  | ... | ... | ... |

  ## 二、架构评估
  ### 优点
  ### 改进建议

  ## 三、性能分析
  ### 瓶颈识别
  ### 扩展性评估

  ## 四、安全审查
  ### 安全风险
  ### 权限控制

  ## 五、可维护性
  ### 代码规范
  ### 文档完整性

  ## 六、风险清单
  | 风险项 | 级别 | 建议措施 |
  |--------|------|----------|

  ## 七、改进建议
  [优先级排序的具体建议]
```
